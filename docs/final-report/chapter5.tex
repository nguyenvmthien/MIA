\chapter{Thành phần Agentic AI}

\section{Kiến trúc agent}

Meeting AI Agent có hành vi agentic ở lớp orchestrator. Thay vì gọi LLM một lần và lưu raw output, orchestrator ra quyết định dựa trên trạng thái trung gian: audio có hợp lệ hay không, transcript có dài quá prompt window hay không, output JSON có đúng schema không, assignee có match roster không và task có đủ evidence để tự động lưu hay cần human review.

\section{Luồng xử lý và pseudocode}

\begin{lstlisting}[language=Python3,caption={Luồng xử lý agentic}]
def run_meeting_agent(audio, roster, meeting_date):
    audio = validate_and_preprocess(audio)
    turns = transcribe_and_diarize(audio)
    chunks = chunk_transcript(turns)

    tasks = []
    for chunk in chunks:
        prompt = build_prompt(chunk, roster, meeting_date)
        raw = llm_generate(prompt)
        parsed = validate_json_schema(raw)
        tasks.extend(parsed)

    checked = apply_guardrails(tasks, turns)
    resolved = resolve_assignees(checked, roster)
    store_results(turns, resolved)
    emit_metrics(resolved)
    return resolved
\end{lstlisting}

\section{Ví dụ quyết định của agent}

Nếu transcript có câu ``Bob, can you send the report by Friday?'' và roster chứa \texttt{Robert Kim} với alias \texttt{Bob}, agent dùng roster như một tool để chuẩn hoá assignee thành \texttt{Robert Kim}. Nếu có nhiều người cùng alias hoặc confidence thấp, task được đánh dấu review để người dùng xác nhận. Đây là decision-making dựa trên intermediate outputs: transcript, roster, LLM output và guardrail signals.

\section{Sử dụng công cụ}

Các tool trong pipeline gồm ASR/diarization, database, Redis queue/cache, Ollama inference, roster resolver, guardrails, Prometheus metrics và feedback exporter. Nhờ tách tool theo trách nhiệm, hệ thống có thể audit từng bước thay vì chỉ lưu câu trả lời cuối cùng của model.
