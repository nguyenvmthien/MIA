\section{Bộ Dữ liệu Đánh giá}

Bộ smoke test (\texttt{data/eval/gold\_smoke.jsonl}) gồm 5 mẫu cuộc họp được annotate thủ công, mỗi mẫu chứa transcript và danh sách action items ground truth. Script \texttt{train/evaluate.py} tính precision, recall và F1 ở cấp độ task description.

\begin{infobox}
\textbf{Định nghĩa Metrics:}
\begin{align}
\text{Precision} &= \frac{|\text{Extracted} \cap \text{Gold}|}{|\text{Extracted}|} \\[4pt]
\text{Recall} &= \frac{|\text{Extracted} \cap \text{Gold}|}{|\text{Gold}|} \\[4pt]
F_1 &= 2 \cdot \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}}
\end{align}
Ngưỡng CI (Continuous Integration): Precision $\geq 0.70$.
\end{infobox}

\section{Hiệu quả của Redis Prompt Cache}

Cache hit rate phụ thuộc vào tần suất lặp lại prompt (tức là cùng cuộc họp được gửi lại). Trong môi trường development, cache giúp giảm latency từ $>$30s xuống $<$10ms cho các cuộc họp trùng lặp, tiết kiệm đáng kể thời gian debug và test.

\section{Phân tích Guardrail Effectiveness}

Trên 5 mẫu smoke test:
\begin{itemize}
  \item \textbf{Schema failure rate = 0:} 100\% LLM output được parse thành JSON hợp lệ.
  \item \textbf{Hallucination flags = 0:} Không có trường hợp nào bị flag là hallucination trên smoke test, cho thấy few-shot prompt với roster injection đủ để constrain output.
  \item \textbf{Jailbreak detections = 0:} Không có mẫu test nào chứa injection patterns (expected).
\end{itemize}
