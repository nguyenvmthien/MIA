\section{Kiến trúc runtime}

Các thành phần chính:

\begin{itemize}[leftmargin=*]
  \item \textbf{Frontend}: thư mục \texttt{web/}, xây bằng Next.js. UI hỗ trợ upload audio, theo dõi trạng thái job, review transcript, speaker mapping, task feedback và roster/history.
  \item \textbf{API}: \texttt{src/meeting\_agent/api/main.py}, cung cấp REST endpoints cho meeting, transcript, feedback, metrics, retrain và deploy.
  \item \textbf{Worker}: Celery worker trong \texttt{src/meeting\_agent/pipeline/worker\_task.py}, xử lý pipeline audio và routing queue.
  \item \textbf{Storage}: PostgreSQL lưu meeting, participants, transcript turns, tasks, artifacts, feedback corrections, calendar events và worker registry.
  \item \textbf{Queue/cache}: Redis dùng cho Celery broker và cache một số bước inference.
  \item \textbf{LLM serving}: Ollama phục vụ Qwen2.5-3B và embedding model nếu được cấu hình.
  \item \textbf{Monitoring}: Prometheus và Grafana.
\end{itemize}

\section{Database-backed workflow}

Các phần trước đây dễ mất trạng thái nay đã được đưa vào database:

\begin{itemize}[leftmargin=*]
  \item Transcript turns thay vì chỉ nằm trong file artifact.
  \item Meeting participants và speaker mapping.
  \item Action items và feedback corrections.
  \item Calendar events để audit việc đồng bộ lịch.
  \item Worker registry và metrics phục vụ dashboard.
  \item Raw artifacts như diarization output và model/prompt metadata.
\end{itemize}

Thiết kế này giúp hệ thống dễ audit hơn và là nền tảng cho feedback export/fine-tuning.

\section{Human feedback UI}

Bước Human Feedback đã được điều chỉnh theo đúng bản chất multi-speaker:

\begin{itemize}[leftmargin=*]
  \item Panel transcript hiển thị lượt nói theo speaker.
  \item Panel speaker evidence cho thấy mỗi \texttt{SPEAKER\_xx} đã nói những đoạn nào.
  \item Người dùng map speaker ẩn danh sang participant thật thay vì hệ thống tự áp một speaker cho toàn bộ audio.
  \item Task card hiển thị evidence turn để người dùng biết tại sao task được trích xuất hoặc gán cho người đó.
  \item Layout review responsive hơn, không bị co lại thành một cụm nhỏ ở giữa màn hình.
\end{itemize}

\section{MLOps services}

Profile \texttt{mlops} trong Docker Compose bổ sung:

\begin{itemize}[leftmargin=*]
  \item \texttt{beat}: Celery Beat schedule cho retrain check và drift check.
  \item \texttt{trainer}: worker riêng chỉ consume queue \texttt{mlops}.
  \item \texttt{mlflow}: tracking server cho experiment khi chạy training.
  \item \texttt{Dockerfile.train}: image tách riêng dependency GPU/training khỏi runtime API.
\end{itemize}

Các lệnh vận hành chính:

\begin{lstlisting}[language=bash,caption={Lệnh vận hành MLOps}]
make mlops-up
make retrain-check
make retrain-force
make train-image
make deploy-promoted-model APPLY=1
\end{lstlisting}
