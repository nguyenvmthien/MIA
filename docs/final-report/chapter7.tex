\section{Calendar và Notification Integration}

Tích hợp Google Calendar API và email notifications để tự động tạo event và gửi reminder từ action items có due\_date:

\begin{lstlisting}[language=Python3, caption={Pseudo-code tích hợp Google Calendar.}]
for task in meeting.action_items:
    if task.due_date and task.assignee_email:
        calendar_api.create_event(
            title=task.description,
            date=task.due_date,
            attendees=[task.assignee_email],
            description=f"Meeting: {meeting.meeting_id}"
        )
        email_client.send_reminder(
            to=task.assignee_email,
            subject=f"Action item: {task.description}",
            body=render_template("task_reminder.html", task=task)
        )
\end{lstlisting}

\section{Cải thiện Mô hình}

\begin{itemize}[leftmargin=*]
  \item \textbf{Nâng cấp LLM:} Qwen2.5-7B hoặc 14B sẽ cho precision và recall cao hơn, đặc biệt trong các cuộc họp kỹ thuật phức tạp. Với bộ nhớ GPU đủ ($\geq$16GB), 7B là lựa chọn tối ưu về chất lượng/chi phí.
  \item \textbf{Multi-turn RAG:} Thay vì per-chunk RAG, xây dựng conversation memory để LLM tham chiếu action items từ chunk trước, giải quyết vấn đề context bị cắt.
  \item \textbf{Tiếng Việt fine-tuning:} Thu thập dữ liệu cuộc họp tiếng Việt, fine-tune với QLoRA trên domain-specific vocabulary (kỹ thuật, kinh doanh).
\end{itemize}

\section{Infrastructure}

\begin{itemize}[leftmargin=*]
  \item \textbf{Kubernetes:} Horizontal Pod Autoscaling cho Celery workers khi load tăng.
  \item \textbf{PostgreSQL:} Thay file JSON bằng database cho worker registry và meeting metadata.
  \item \textbf{Distributed training:} Ray hoặc DeepSpeed cho fine-tuning cluster GPU.
  \item \textbf{Model serving:} vLLM hoặc TensorRT-LLM thay Ollama cho throughput cao hơn 5--10×.
\end{itemize}
