\chapter{Hướng phát triển}
\label{chap:future}

\section{Mở rộng dữ liệu}

Ưu tiên tiếp theo là thu thập thêm feedback thật từ UI, mở rộng gold set và thêm dữ liệu tiếng Việt/đa ngôn ngữ. Khi số correction đạt ngưỡng, có thể chạy fine-tuning trên GPU và so sánh với baseline hiện tại.

\section{Nâng cấp evaluation}

Hiện tại evaluation dựa trên matching action item ở mức mô tả. Hướng phát triển là thêm semantic matching, đánh giá riêng assignee/deadline/priority, đo hallucination rate và báo cáo confidence calibration.

\section{Production hardening}

Nếu triển khai thật, hệ thống cần:

\begin{itemize}[leftmargin=*]
  \item HTTPS/reverse proxy và secret manager.
  \item Backup/restore PostgreSQL và object storage cho artifacts.
  \item Role-based access control chi tiết hơn.
  \item Alert rules cho queue backlog, worker failure, API latency và GPU memory.
  \item Retention policy/cleanup cho raw artifacts.
\end{itemize}

\section{Tự động hoá release có kiểm soát}

Serving switch hiện là manual. Trong tương lai có thể thêm release policy bán tự động: model promoted được deploy vào canary, so sánh A/B trong một khoảng thời gian, sau đó mới chuyển traffic chính nếu không có regression.
